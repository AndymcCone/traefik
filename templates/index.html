Mikä on docker

Sovelluksia kehitetään ja testataan usein yhdellä koneella. Tämä johtaa ongelmaan, joka tunnetaan yleisemmin nimellä "toimii koneellani", jossa Kehittäjä ei tiedä miksi tai miten sovellus ei toimi toisella koneella. Lisäksi järjestelmän eri osat voivat muuttua ajan myötä, mikä saattaa johtaa siihen, että sovellus ei toimi. Nämä muutokset voivat olla mitä tahansa käyttöjärjestelmän päivityksestä riippuvuuksien muutoksiin tai jopa laitteiston muutoksiin. Docker on kokoonpanonhallintatyökalu, jota käytetään ohjelmistojen käyttöönoton automatisoimiseen kevyissä säiliöissä. Nämä säiliöt auttavat sovelluksia toimimaan tehokkaasti eri ympäristöissä. Docker yhdistää sovelluksen ja sen riippuvuudet kuvaksi, jota voidaan sitten käyttää millä tahansa koneella, jos se pystyy ajamaan Dockeria.

Dockerin ominaisuudet:

  • Helppo ja nopea konfigurointi
  • Sovelluksen eristäminen
  • Turvallisuuden hallinta
  • Korkea tuottavuus
  • Korkea skaalautuvuus
  • Infrastruktuurista riippumaton
Docker-säiliö on ohjelmistopaketti, jossa on kaikki sovelluksen suorittamiseen tarvittavat riippuvuudet. Docker yhdistää sovelluksen ja sen riippuvuudet kuvaksi, jota voidaan sitten käyttää millä tahansa koneella, jos se pystyy ajamaan Dockeria.

Docker Compose

Alusta alkaen olemme jo käsitelleet lukuisia komentorivivaihtoehtoja sekä kuvan rakentamisessa, työntämisessä että suorittamisessa. On kuitenkin olemassa näppärempiäkin vaihtoehtoja, kuten docker-compose-niminen työkalu näiden hallitsemiseksi. docker-compose on suunniteltu yksinkertaistamaan monisäilösovellusten suorittamista yhden komennon käyttämiseen. Composen avulla voimme luoda YAML-tiedoston palveluiden määrittelemiseksi ja yhdellä komennolla kääntää tai purkaa kaiken.

Dockerfile

Dockerfile on yksinkertainen tekstitiedosto, jossa on joukko komentoja tai ohjeita. Nämä komennot/ohjeet suoritetaan peräkkäin toimintojen suorittamiseksi peruskuvalle uuden telakointikuvan luomiseksi. kommentit ja komennot + argumentit ovat kahdenlaisia ​​päärivilohkoja Dockerfile-syntaksissa Peruskomennot FROM – Määrittää käytettävän peruskuvan ja käynnistää rakennusprosessin. RUN – Se suorittaa komennon ja sen argumentit kuvasta. CMD – Samanlainen toiminto kuin RUN-komento, mutta se suoritetaan vasta, kun säilö on instantoitu. ENTRYPOINT – Se kohdistaa oletussovellukseesi kuvassa, kun säilö luodaan. ADD – Se kopioi tiedostot lähteestä kohteeseen (säilön sisällä). ENV – Asettaa ympäristömuuttujat.
